# Bangumi 每日自动发文模块测试报告

## 测试日期

2026-04-09

## 测试范围

### 单元测试
- 候选条目筛选
- 封面图候选评分与优先级
- Markdown 文章模板生成
- 状态记录生成

### 集成验证
- 工作流已接入 `bangumi-update.yml`
- 文章输出目录与内容 schema 兼容
- 审核模式与正式发布模式可切换
- 自动提交范围已包含缓存、文章、状态文件、报告文件

## 重点验证结论

1. **定时集成**：新模块已接入现有 Bangumi 每日固定时间自动更新流
2. **正文三图**：生成模板会自动随机插入 3 张关联图片
3. **封面图兜底**：当网络图片搜索失败时，使用默认封面
4. **结构统一**：生成文章采用站内长文栏目结构
5. **去重有效**：通过 `src/data/bangumi-posts-state.json` 与现有文章索引双重去重
6. **审核支持**：支持 `draft: true` 预览/审核模式

## 风险与说明

- 自动生成正文目前基于 Bangumi 条目详情与公开简介整理，属于“资料型介绍文”，可读性稳定，但深度仍低于完全人工撰写文章
- Jikan 图片回退依赖公网访问，若第三方接口不可用，会自动退回站内默认封面
- 文章结构已保持统一，但具体文风仍建议在试运行阶段观察几天后再微调

## 建议上线策略

### 第一阶段
- 打开 `BANGUMI_POST_REVIEW_MODE=true`
- 连续观察 3~7 天
- 审核生成文章质量和封面命中率

### 第二阶段
- 切换 `BANGUMI_POST_REVIEW_MODE=false`
- 保持 `BANGUMI_POSTS_PER_RUN=1`
- 稳定后再考虑提高每日生成数量
